## 1. Executive summary

`auto-research` is a two-plane multi-agent research platform for cross-asset language-driven alpha in AI infrastructure and frontier-tech equities. It is the engineering corrective to the popular but architecturally flawed `virattt/ai-hedge-fund` repo: rather than placing the LLM at the trading-decision layer (where it adds noise), it places the LLM where it is uniquely valuable — unstructured-to-structured feature extraction from corporate disclosures and agentic research loops that propose, validate, and document new signals.

The system reads 10-K, 10-Q, 8-K, S-1/S-3 SEC filings and earnings call transcripts on ~90 AI-related equities; extracts structured language features (guidance tone, evasiveness, supplier mentions, milestone/dilution disclosures) via a deterministic LLM extraction plane; combines them into three signal families (cross-doc supply-chain forward-tone propagation, PEAD-style language drift, frontier-tech milestone/dilution); validates them through a tiered analytical gauntlet (information-content tests → portfolio backtest with CPCV + deflated Sharpe → stress tests); and exposes an agentic research loop and a daily adverse-selection critic, both implemented in LangGraph / Pydantic AI.

The architecture is faithful to the v3 design document's Figure 1: the LLM never sits in the trading hot path; extraction is nightly batch with content-hash idempotent caching; research is asynchronous; the live critic is a haircut, never a directional override.

## 2. Goals and non-goals

### Goals

- **Interview portfolio anchor.** A defensible, demoable system that maps to JD bullets for AI Engineer, ML Engineer, and Quant Researcher roles simultaneously.
- **Self-learning vehicle.** Hands-on implementation of 2026-current best practices across LLM ops, ML ops, agentic systems, RAG, feature stores, and quantitative research validation.
- **Plausible real-alpha aspiration.** Real signals tested rigorously against the López de Prado validation discipline — the architecture is honest enough that an interviewer cannot dismiss it as demo theatre.
- **Defensible architectural judgment story.** Every choice (LLM placement, build-vs-reuse, validation tier, tool selection) traces to a one-sentence justification rooted in the source design doc.

### Non-goals

- Live capital deployment. Paper portfolio only in v1. Execution-model is realistic enough to support live trading later but no broker integration in scope.
- Multi-asset reach beyond equities. Crypto / prediction markets / FX are explicitly out — they are documented as extension points but not built.
- Black-Litterman combiner. Documented as a "next step" but not implemented. v1 uses IC-weighted combiner with Ledoit-Wolf shrinkage.
- Investor-persona LLM scorers. The whole point of the project is to *not* re-skin the v0 anti-pattern. Permanently excluded.

## 3. Constraints

- **Universe:** ~90 names across two sub-universes (~70 AI infra supply chain + ~20 frontier tech speculative).
- **Backfill window:** 2 years (Jan 2024 – present). Matches the AI-capex regime onset; CPCV with embargo gives ~40-60 walk-forward folds at weekly frequency.
- **Hardware:** local macOS. No cloud GPU. Sentence-transformer embeddings run CPU-only.
- **External services:** Anthropic API (required), OpenAI Whisper API (transcripts; FMP was originally planned, deferred per ADR `docs/decisions/2026-05-23-transcripts-source.md`), Voyage AI (embeddings, optional fallback to local BGE), EDGAR (free), LangSmith / Langfuse self-hosted via Docker.

## 6. Data layer

### 6.1 Ingest (`src/auto_research/ingest/`)

- `edgar.py` — SEC EDGAR public JSON + full-text-submission endpoints for 10-K, 10-Q, 8-K, S-1, S-3. Idempotent on `(cik, accession_number)`. Records `accepted_datetime` as the canonical point-in-time stamp.
- `transcripts/` — earnings-call transcript ingest. Audio is fetched via per-platform sources under `transcripts/sources/` (direct MP3 from issuer IR pages where available, otherwise YouTube via yt-dlp against aggregator uploads), then transcribed via the OpenAI Whisper API in `transcripts/_whisper.py`. Records `event_datetime` (call start). Coverage gaps flagged with `status="no_coverage"` in the manifest; transient infrastructure failures get retryable `status="error"`. Per-ticker source routing + query overrides live in `config/transcripts/sources.toml`. Decision rationale + source-platform trade-offs in `docs/decisions/2026-05-23-transcripts-source.md`.
- `manifest.py` — append-only Parquet ledger keyed on `(source, doc_id)`. Schema: `source`, `entity_id`, `doc_id`, `form_type`, `event_datetime` (the source-specific PIT stamp; e.g. EDGAR `acceptanceDateTime`), `fetched_at`, `content_sha256`, `path`, `status`. Re-runs are no-ops on `(source, doc_id)` hit with `status='ok'`; rows with non-`ok` status remain eligible for retry. Writes are atomic (tmp + `os.replace` + fsync) and serialised across processes via `fcntl.flock`; the lock-holding `append` also dedups incoming rows against the on-disk ledger so the snapshot-then-write race can't double-insert. Schema changes require an explicit migration step — there is no in-code promote / forgive layer.

### 6.2 Raw document store (`data/raw/`)

Filesystem partitioning: `data/raw/{source}/{entity_id}/{year}/{doc_id}.{ext}`. Preserves original bytes. Source for both extraction and reproducibility audits.

### 6.3 PIT feature store — Feast on Parquet (`feast/`)

| Component | Choice |
|-----------|--------|
| Registry | Feast `0.43+` local SQLite |
| Offline store | File-based Parquet |
| Online store | SQLite (disabled — offline-only for v1) |
| Entity | `entity_id` (ticker) |
| FeatureViews | `ten_k_features`, `transcript_features`, `eight_k_features`, `s_filing_features`, `price_features`, `signal_features` |
| FeatureServices | `signal_a1`, `signal_a2`, `signal_b1` |

Every row carries `event_datetime` (publication time) and `as_of_ts` = `event_datetime + lag-1 trading day cutoff` baked in at write-time. **Lag-1 is structural, not enforced by query convention.** The PDF's "single most common silent killer" lookahead failure mode is architecturally impossible.

### 6.4 Extracted store (`data/extracted/`)

Raw Pydantic LLM outputs persisted as JSON Lines, partitioned by `(worker, year, month)`. Audit trail and reproducibility kernel — never queried in the trading path.

## 7. Extraction plane

### 7.1 Workers

| Worker | Source | Primary features | Used by signal |
|--------|--------|-----------------|----------------|
| `ten_k.py` | EDGAR 10-K | guidance tone, accrual flags, supplier/customer mentions w/ source spans, language novelty vs prior 10-K, risk-factor deltas | A1, A2 |
| `transcript.py` | Whisper transcripts of IR audio | prepared-remarks tone, Q&A evasiveness, forward statements with entity links + horizons | A1, A2 |
| `eight_k.py` | EDGAR 8-K | event classification, milestone / partnership / contract mentions, dilution-language flags | A1, B1 |
| `s_filings.py` | EDGAR S-1, S-3 | dilution event detection, capital-raise language, use-of-proceeds | B1 |

Each worker is a stateless `(raw_doc, prompt_version) → ExtractionOutput` pure function with content-hash idempotent cache.

### 7.2 Pydantic schemas (`src/auto_research/extract/schemas.py`)

All extracted claims carry `source_span: tuple[int, int]` and `source_quote: str` — verbatim source text supporting the claim. A post-validation step asserts `source_text[span[0]:span[1]] == source_quote`. Hallucinations are caught at write-time.

### 7.3 Tiered model routing

| Worker / task | Model | Why |
|---------------|-------|-----|
| `ten_k.py` cross-doc supplier mapping; Q&A evasiveness in `transcript.py` | Sonnet 4.6 | Cross-doc reasoning + nuanced language judgment |
| `eight_k.py` event classification, `s_filings.py` dilution-language | Haiku 4.5 | Pattern recognition, high volume |
| Routine fields (guidance tone, accrual flags) | Haiku 4.5 | Templated extraction |
| Research agent / live critic | Sonnet 4.6 default, Opus 4.7 for hard critiques | Multi-turn reasoning |

### 7.4 Backfill economics

~90 names × ~15 docs/yr × 2 yr ≈ 2,700 docs. Weighted avg ~30K input tokens / doc. **With Batch API (50% off) + prompt caching (~80% of input cached at $0.30/M):**

Backfill total ≈ **$75-150 one-time**. Daily incremental ~$0.05-0.20.

### 7.5 Guardrails (`src/auto_research/extract/guardrails.py`)

- **Guardrails AI** validators: PII detection, numeric-range sanity, monetary-amount sanity, length bounds.
- **Custom citation-grounding validator**: verifies every `source_quote` appears verbatim in source text, and an LLM-judge asserts the quote semantically supports the extracted claim.
- **Pydantic schema validation** on every extraction. Failures route to `data/quarantine/` for human review — never silently retried with degraded data.

## 8. RAG layer

RAG is first-class. Four distinct flows, each with its own approach.

### 8.1 Flow 1 — Long-doc extraction (`extract/chunking.py`, `extract/rag_retrieval.py`)

Hybrid policy:

- **Docs < 100K tokens** (all 10-Q, 8-K, S-1/S-3, transcripts, ~80% of 10-Ks) → single-shot extraction with prompt caching. Faster, cheaper, preserves cross-section coherence.
- **Docs ≥ 100K tokens** (long 10-Ks) → section-aware chunking + contextual chunking + retrieval-augmented extraction.

Stack:
- `unstructured.io` parses 10-K, respecting Item 1A / 7 / 8 boundaries.
- Anthropic's **contextual chunking** pattern: each chunk gets an LLM-generated context line prepended (e.g., *"This chunk is from NVDA Q3-2025 10-Q MD&A discussing China export controls"*). ~50% retrieval lift per Anthropic's 2024 paper. Generation calls cached.
- Chunks embedded with Voyage `voyage-3` (or local `bge-small-en-v1.5` fallback).
- Per-doc LanceDB store.
- For each Pydantic schema field, retrieve top-k relevant chunks → extraction call from retrieved context.

### 8.2 Flow 2 — Memo retrieval (`agents/memo_retrieval.py`)

Used by research agent's `propose_hypothesis` to ground new hypotheses in past memos (especially kill memos — "what have I already tried?").

- LanceDB at `data/rag/memos.lance`.
- Embeddings: Voyage `voyage-3` primary, `bge-small-en-v1.5` local fallback.
- **Hybrid retrieval**: BM25 (`rank_bm25`) + dense + RRF (Reciprocal Rank Fusion) merge.
- **Reranking**: `bge-reranker-base` (local) on top-20 → top-5 final.
- Memos embedded at write-time as part of `write_memo` node.

### 8.3 Flow 3 — Entity resolution (`extract/entity_resolution.py`)

Critical for signal A1. Maps fuzzy mentions ("our optical interconnect partners," "leading networking ASIC supplier") in narrative-source docs to specific tradeable tickers.

- Universe entities + aliases (e.g., `{"CRDO": ["Credo Technology", "Credo Semiconductor"]}`) embedded once with Voyage.
- Mention text → dense similarity to universe → top-3 candidates → LLM disambiguator picks or returns `unknown`.
- Disambiguator stores reasoning for audit.

### 8.4 Flow 4 — Literature retrieval (deferred to stretch)

Designed-for, not built in v1. Extension point at `agents/literature_retrieval.py` with a stub. Future implementation: Tavily web search or local PDF corpus of foundational papers (Tetlock 2007, Loughran-McDonald 2011, López de Prado 2018) parsed with `unstructured.io` and queryable via the same LanceDB infrastructure.

### 8.5 RAG eval — Ragas pytest integration

```python
# tests/evals/test_rag_retrieval.py
@pytest.mark.rag_eval
def test_memo_retrieval_recall():
    results = ragas.evaluate(
        dataset=memo_retrieval_eval_set,
        metrics=[context_precision, context_recall, faithfulness, answer_relevancy],
    )
    assert results["context_recall"] > 0.75
    assert results["faithfulness"] > 0.85
```

Hand-built eval set: ~30 `(query, expected_memo_ids)` pairs for Flow 2; ~30 `(question, expected_chunk_ids)` pairs for Flow 1. CI gate alongside DeepEval extraction evals.

OpenInference RAG span conventions (`RETRIEVER`, `RERANKER`) used for tracing — Langfuse displays the retrieval chain visually.

## 9. Signal library

### 9.1 A1 — Hyperscaler forward-tone propagation (cross-doc, LLM-unique)

For each tradeable name T on day d: forward-demand index = time-decayed-weighted sum of mentions of T (or its sector) extracted from narrative-source docs in trailing 60 days. Cross-sectional rank → top-quintile long, bottom-quintile short, sector-neutralized within tradeable sub-book.

Source features: `transcript_features.supplier_mentions`, `ten_k_features.supplier_mentions`. Entity resolution (Flow 3) maps mention → ticker.

### 9.2 A2 — PEAD-flavored language drift (academically grounded)

Post-earnings window of 5–20 trading days. High-confidence + low-novelty → long. High-evasiveness + high-novelty → short. Per-name signal, not cross-sectional.

Source features: `transcript_features.q_and_a_evasiveness`, `ten_k_features.guidance_tone`, `ten_k_features.language_novelty_score`.

### 9.3 B1 — Frontier-tech milestone / dilution

5-day windows. Positive milestone language (partnership, contract, FDA-equivalent approval) → long. Dilution-language detection on S-3/S-1 → short.

Source features: `eight_k_features.event_classification`, `s_filing_features.dilution_flags`.

### 9.4 Combiner (`signals/combiner.py`)

IC-weighted, with Ledoit-Wolf shrinkage on the signal-correlation matrix. Per-name daily alpha score. No LLM aggregation.

## 10. Backtest gauntlet — tiered validation

### 10.1 Tier 1 — Information analysis (`backtest/info_tests.py`)

First-pass screening. Seconds-to-minutes runtime. Pure info-content hypotheses can stop here.

- `event_study(events_df, returns_df, window)` — CAR ± window + bootstrap CI
- `ic_analysis(features_df, fwd_returns_df, horizons)` — Spearman IC by horizon + t-stat + decay curve
- `quantile_sort(features_df, fwd_returns_df, n_quantiles)` — mean return per quantile + top-bottom + t-stat
- `conditional_distribution(features_df, fwd_returns_df, condition)` — lift, hit-rate, posterior shift
- `mutual_information(features_df, fwd_returns_df)` — KSG estimator
- `bootstrap_significance(stat_func, data, n_boot)` — generic bootstrap CI

### 10.2 Tier 2 — Portfolio backtest

Built on **vectorbt.pro** for generic mechanics + custom layers:

- `labels.py` — triple-barrier labels (López de Prado) with vol-adjusted bands
- `cpcv.py` — combinatorial purged CV with embargo (~50-60 folds at weekly frequency)
- `deflated_sharpe.py` — multiple-testing-adjusted Sharpe
- `costs.py` — FMP bid-ask half-spread + sqrt impact + IBKR borrow proxy + commissions, plumbed into `vbt.Portfolio.from_signals`
- `engine.py` — daily long-short, sub-universe-neutralized, vol-scaled to 10% target portfolio vol, single-name cap 5%, turnover penalty, ADV-based participation cap

### 10.3 Tier 3 — Stress tests

Regime breakouts, 2x cost stress, hyperparameter sensitivity, decorrelation against existing alpha library.

### 10.4 `BacktestReport` and `InfoReport` dataclasses (`backtest/report.py`)

Structured reports with IC stats, performance (cost-adjusted), robustness breakdown, risk decomposition, cost sensitivity. Persisted as MLflow artifacts.

### 10.5 Tier-aware decide gates (`agents/research_graph.py`)

```python
T1_GATE = {
    "ic_t_stat_min": 2.0,
    "top_minus_bottom_t_stat_min": 1.8,
    "event_study_car_t_stat_min": 2.0,
    "n_observations_min": 30,
}

T2_GATE = {
    "deflated_sharpe_min": 1.0,
    "ic_mean_min": 0.02,
    "ic_half_life_min_days": 1.0,
    "capacity_usd_min": 1_000_000,
    "sharpe_net_min": 0.7,
    "sharpe_at_2x_costs_min": 0.3,
    "max_beta_to_existing": 0.5,

```

Promote / iterate / kill is **code-checked, not LLM judgment**. LLM critic runs in parallel and produces qualitative addendum to the memo.

## 11. Research agent (`src/auto_research/agents/research_graph.py`)

LangGraph state machine. The interview showcase.

### 11.1 Hypothesis types

| Type | Cost | Example |
|------|------|---------|
| Feature-extraction | High (re-extract + re-materialize Feast) | "Add capex-language-confidence to 10-K extraction" |
| Conditional / interaction | Low (DuckDB query) | "A2 evasiveness only when recent S-3 dilution-flag present" |
| Event-window | Low | "A1 with 10-day delay before entry on names with high short-interest" |
| Regime-conditional | Low | "B1 gated by sector momentum quintile" |
| Cross-signal interaction | Low | "Long A1 ∩ A2 confluence only" |
| Pure info-content | Low — stops at T1 | "Does evasiveness predict implied-vol expansion?" |

### 11.2 Graph nodes

```
propose_hypothesis (Researcher, conditioned on memo retrieval)
    ▼
materialize_signal (feature route OR signal-def route)
    ▼
run_validation (tier-routed: T1 / T2 / T3)
    ▼
┌────────┴────────┐
decide (code-gate)  critique (Critic LLM, parallel)
└────────┬────────┘
    ▼
HITL interrupt (human approves promote)
    ▼
write_memo (always — promote / iterate / kill all produce memos)
```

### 11.3 LangGraph harness

- **Checkpointer:** `langgraph.checkpoint.sqlite.SqliteSaver` for resumability + replay.
- **HITL interrupt:** `interrupt()` at `decide_promote` for human approval before library promotion.
- **Replay:** `uv run auto-research research replay <session_id>` reruns deterministically.
- **Streaming:** node-by-node streaming for live agent traces in Langfuse.

### 11.4 Tools via MCP

Research agent invokes all read-only data + research tools via an MCP server (see §13).

## 12. Live critic (`src/auto_research/agents/live_critic.py`)

**Built with Pydantic AI** (not LangGraph) — demonstrates framework-judgment by picking tool per task.

Pipeline: `fetch_context (news + positions) → flag_overhangs → assess_haircut → emit_haircut_vector`.

Haircut ∈ [0, 1] multiplied into position sizing. Never a directional override. PDF page 5 distinction respected.

Runs daily via cron (`scripts/cron_daily_critic.sh`).

### 13.2 Reliability primitives (`src/auto_research/agents/reliability.py`)

Decorators / context managers:
- Circuit breaker (N consecutive failures → stop)
- Cost cap per agent session (counted from Anthropic API usage metadata, hard $ limit)
- Max-iteration guard (10 cycles max through propose→backtest→decide)
- Exponential backoff + jitter retry (capped at 3 retries on 5xx / rate limit)
- Fallback model (Sonnet rate-limited → Haiku with trace flag)

> **Superseded by `docs/CONTRACTS.md` §5 (Issue #8 review):** the
> max-iteration guard moved to LangGraph's `recursion_limit` (counts node
> transitions across the whole traversal; the right dimension for a
> graph cycle cap) and the fallback-model decorator was dropped (silent
> downgrade hides capacity signal, changes the producing model, and
> complicates cost accounting). The contract surface is now three
> primitives: cost cap, circuit breaker, retry with backoff.

## 14. Eval

### 14.1 Extraction quality

- **Gold sets:** ~50-80 hand-labeled examples per worker at `eval/gold_sets/{worker}.jsonl`.
- **DeepEval pytest suite** at `tests/evals/test_{worker}_extraction.py`:
  - Reference-based: F1 / exact-match / Spearman per Pydantic field
  - G-Eval (LLM-judge) for subjective fields (evasiveness, forward_tone_quality)
  - Hallucination metric: does the extracted claim appear in source doc?
- **Continuous monitoring:** every nightly extraction logs aggregate pass rates to MLflow. >5pp regression vs baseline → run flagged.

### 14.2 RAG quality

- **Ragas pytest suite** at `tests/evals/test_rag_*.py`:
  - Flow 1 (long-doc extraction): context_precision, context_recall, faithfulness, answer_relevancy
  - Flow 2 (memo retrieval): context_precision, context_recall

### 14.3 Research-agent memo quality

- **G-Eval rubric** on memo faithfulness (does the memo's claims trace to the BacktestReport?), traceability (every claim references a memo or doc), completeness (covers gate results + qualitative critique).

### 14.4 Backtest validity

Covered by tiered gates (§10.5). No additional eval needed — the gates *are* the eval.

## 16. LLM cost stack

- **Anthropic API + Batch API (50% off) + prompt caching ($0.30/M cached input).**
- **Tiered models:** Haiku (high-volume templated extraction), Sonnet (cross-doc reasoning + agent default), Opus (hard critic moments only).

### 23.4 Branch + worktree workflow

- Main checkout at `~/Documents/projects/auto-research/` stays on `main`.

- Enables parallel Claude Code sessions on independent issues — particularly useful in W2-W3 where RAG / signals / backtest can advance concurrently.
- Cleanup on PR merge: `git worktree remove ../auto-research-trees/issue-N-*`.

## 24. Glossary

- **PIT** — Point-in-time. Data discipline ensuring features are valid only after their publication timestamp + lag.
- **CPCV** — Combinatorial Purged Cross-Validation. López de Prado's CV for financial time-series with embargo to prevent label leakage.
- **Deflated Sharpe** — Sharpe ratio adjusted for multiple-testing across signal hypotheses and parameter sweeps.
- **Triple-barrier labels** — López de Prado label scheme: upper / lower / time barriers determine outcome label ∈ {-1, 0, +1}.
- **IC** — Information Coefficient. Spearman rank correlation between signal value and forward return.
- **RRF** — Reciprocal Rank Fusion. Canonical method for merging hybrid retrieval result sets.
- **PEAD** — Post-Earnings Announcement Drift. Documented anomaly where returns continue in earnings-surprise direction for weeks afterward.
- **MCP** — Model Context Protocol. Anthropic's standard for exposing tools and resources to LLM clients.
- **HITL** — Human-In-The-Loop. Pattern where automated agent pauses for human approval at critical decisions.
- **G-Eval** — DeepEval's LLM-as-judge metric with custom rubrics.
- **RAG** — Retrieval-Augmented Generation. LLM grounded on retrieved context rather than parametric memory.
